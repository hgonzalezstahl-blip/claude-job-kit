# Build Your Own (level up)

The kit comes with assistants someone else set up for you. The real power of Claude Code is making your *own* — assistants and memory tuned to exactly what you do. This is how you go from borrowing a setup to building one.

When you're comfortable with the basics, open Claude Code in this folder and paste the prompt below. It teaches you by building one real thing with you, not by lecturing.

---

```
I want to start building my own custom Claude Code setup, the way an experienced user would. I'm a beginner, so teach me as we go, in plain English, one step at a time, and build one real, working thing with me today instead of just explaining theory.

Here's how I'd like you to guide me:

1. First, explain simply what these three things are and why they're useful, using an example from my own life or work for each:
   - an "agent" (a specialist assistant with its own instructions)
   - a "skill" (a reusable rule or procedure)
   - a personal knowledge file (notes you read so you remember things about me and my world)

2. Ask me about the things I do over and over, or the topics I wish you just already knew about me (my industry, my contacts, my projects, my preferences). Pick ONE good candidate with me.

3. Build my first custom agent for that one thing. Create the file in .claude/agents/, write its instructions with me, and explain each part as you go so I understand what I'm looking at. Keep it simple and real.

4. Start a simple personal-knowledge folder (plain markdown files) for things I want you to remember going forward. Help me write the first one or two notes, and explain how you'll use them later.

5. Show me how to actually use what we just built, with a real example.

6. Tell me one small, concrete next thing I could build the next time I sit down, so I keep growing this on my own.

Don't overwhelm me. One real agent and one real note today is a win. Go.
```

---

## A note from whoever set this up for you

Everything in this kit started as one file someone wrote, then improved a little each time it got something wrong. That's the whole trick. Don't try to build a big system. Build one small useful thing, use it, fix what annoys you, repeat. In a few weeks you'll have your own arsenal.
