# Prerequisites and Tool Setup (read this first)

This workspace drafts everything for you, but two tools must be installed or connected once, or you will see errors like Word documents failing, "please install Python," or the LinkedIn search not working. Do these one time and the errors go away.

---

## 1. Python + python-docx (this is what makes the Word resume and cover-letter files)

Why: the polished `.docx` resumes and cover letters are generated by a small Python library called **python-docx**. If Python is not installed, your AI cannot create the Word files and will keep asking you to install it.

### Windows
1. Go to https://www.python.org/downloads/ and download the latest Python. Do NOT use the Microsoft Store version, it has a broken stub that causes errors.
2. Run the installer. On the FIRST screen, check the box that says **"Add python.exe to PATH"**, then click **Install Now**.
3. Open a new PowerShell window and confirm it works: `py --version` (it should print Python 3.x).
4. Install the document library: `py -m pip install python-docx`
5. Confirm the library works: `py -c "import docx; print('python-docx OK')"`

### Mac
1. Check if Python 3 is there: `python3 --version`. If missing, install from python.org or run `brew install python`.
2. Install the document library: `python3 -m pip install python-docx`
3. Confirm: `python3 -c "import docx; print('python-docx OK')"`

Tip: tell your AI once: "On Windows use the `py` launcher (on Mac use `python3`); python-docx is already installed." That stops it from getting confused about how to run Python.

## 2. Apify - the LinkedIn jobs scraper (this is what finds the jobs)

Why: this pulls fresh job postings from LinkedIn (title, company, location, and the full job description) so your AI can tailor your resume to real openings.

1. Create a FREE account at https://apify.com. The free tier is plenty; a job scrape costs only a few cents.
2. In Claude Code, type `/mcp` and connect the **Apify** server. Follow the prompts and sign in / authorize Apify in your browser when asked.
   - If you do not see Apify listed, just ask your AI: "help me connect the Apify MCP server," and follow its steps.
3. Once connected, the LinkedIn scraper is the Apify actor named `curious_coder/linkedin-jobs-scraper`.

### How to use the LinkedIn scraper (you can paste this section to your AI)
- It takes one or more **public** LinkedIn job-search URLs. To make one: open linkedin.com/jobs in an **incognito** window, search your role + location + filters, then copy the URL from the address bar.
- URL filter cheat sheet:
  - `keywords=Director%20AI` (encode spaces as `%20`)
  - `location=Houston%2C%20Texas%2C%20United%20States` (or use `United%20States` plus `f_WT=2` for Remote)
  - `f_E=4%2C5%2C6` (experience level: 4 = mid-senior, 5 = director, 6 = executive)
  - `f_TPR=r604800` (posted in the last 7 days; `r1209600` = 14 days)
- **Target jobs posted in the last 2 weeks or less.** Anything older already has a huge applicant pile, so it is usually too late.
- **The `count` setting is a GLOBAL cap across ALL the URLs in one run, not per URL.** If you pass several search URLs, set `count` high (for example 300) or you will only get a handful of results.
- The results include the FULL job description inline, so your AI can tailor your resume immediately with no extra step.
- LinkedIn public search usually hides salary. Have your AI look up a market range on Glassdoor or levels.fyi.

## 3. Optional but recommended: more job sources

- **Indeed** and **ZipRecruiter** connectors (in your AI's connector settings) add more job sources. Indeed shows real posted salary. ZipRecruiter's salary numbers are unreliable estimates, so ignore Zip comp.
- **Gmail** is the highest-signal source of all, because your own job-alert emails come from searches you already tuned. Connect Gmail so your AI can read and rank those alerts. This catches great roles that keyword scrapes miss (for example "Program Manager, Product Operations" roles that do not have "AI" in the title).

## How the whole thing works (so it is not confusing)

- **The AI drafts (automatic):** reads job alerts, scores how well you fit, and writes the tailored resume, cover letter, application answers, and a networking message.
- **Your computer formats (needs Python):** turns the resume into a polished Word `.docx` using your template. This is the part that needs Python + python-docx from Step 1.
- **You act (manual, on purpose):** you click "submit" on the company site, and you send the LinkedIn messages / ask for referrals yourself. There is no auto-apply, and for senior roles having a human submit is a feature, not a bug.

## Hard-won tips (from real use)

- **Quality over volume.** A few well-targeted applications WITH a warm introduction beat 50 cold applications. A referral is 5 to 10 times more likely to land an interview, and it skips the resume-screening filters.
- **For each target job, find one person you know at the company** (or a 2nd-degree connection) and go through them. Ask your AI to draft the intro request.
- **Message the hiring manager directly** on LinkedIn with a sharp three-sentence note. It leapfrogs the application pile.
- **Match the cover letter font to your resume font.** Open the resume, check the actual font it uses, and make the cover letter match. Do not assume.
- **Keep one master CV** with every role and metric written out in full. Every tailored resume is cut down from it; nothing is invented.
- **Track outcomes** (applied, screen, interview) so you can see what is working and do more of it.
